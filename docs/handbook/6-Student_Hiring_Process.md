# Student Hiring Process

This page outlines how new student employees are hired into the DataSquad, including required forms, timelines, approvals, and communication steps. It is intended for supervisors and project managers who prepare hiring packets and coordinate with Library HR.

---

## Before Hiring

1. Confirm availability of funding (FAU)
2. Confirm position title and rate with Tim
3. Draft a short description of duties for the role
4. Identify the primary and backup supervisors
5. Notify Library HR that a student hire is coming

---

## Required Information

Before starting the hiring request, gather:

- Student's full legal name (LAST, FIRST MIDDLE)
- UID
- g.ucla.edu email address
- Start date (must be firm)
- Working title
- Hours per week
- Hourly rate
- FAU string
- On-campus address and mail code
- Supervisor names and extensions

---

## Steps for Initiating a Hire

1. Submit a Library Student Job Description form  
2. Email Library HR with:
   - Completed form  
   - Student contact information  
   - Planned start date  
3. HR generates the student's hiring packet  
4. Student completes required onboarding steps with Library HR  
5. HR confirms eligibility and provides a formal start date

---

## After HR Confirms

Perform these setup steps:

- Add student to **Google Shared Drives**
- Add to **ucla-data-science-center** GitHub org
- Add to **UCLA-DataSquad** GitHub org
- Add to Slack channels
- Provide DSC Omnilock access
- Add to internal project boards
- Ensure student creates their **Onboarding Issue**  
  <https://github.com/ucla-data-science-center/DataSquad/issues/new/choose>

---

## Off-Boarding Procedures

When a student leaves:

- Remove from GitHub organizations
- Remove from Shared Drives
- Remove Omnilock access
- Remove Slack access
- Archive their work logs
- Ensure all active projects are transitioned
